Specifying Additional Selenium Capabilities

SeLion provides multiple ways to specify additional selenium capabilities.

Via the Annotation

Additional Selenium capabilities can be configured via the SeLion annotations WebTest and MobileTest. When you do this, the additional capabilities provided are local to a single test. For example:



  
Via a Custom Capabilities Provider

  
Additional Selenium capabilities can be configured via a custom capabilities provider. When you do this, the additional
    capabilities are considered for the entire test suite and based on your needs, may or may-not be sent with a single test.
    To create your own capabilities provider you need to build a java class which extends DefaultCapabilitiesBuilder
    and override the method getCapabilities.

In the above example, the capability capName will be added for all tests.

  
Once you create your own custom capabilities provider, you need to tell SeLion to use it. This is accomplished via
    configuration by specifying the system property or environment variable SELION_SELENIUM_CUSTOM_CAPABILITIES_PROVIDER
    or via the TestNG <parameter> customCapabilities. For example:

  
mvn test -DSELION_SELENIUM_CUSTOM_CAPABILITIES_PROVIDER="com.mycompany.test.UserDefinedCapabilities"

  


  
If more than one custom capabilities providers are required, please separate the fully qualified class names with commas.
