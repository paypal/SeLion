Code Generator Maven Plugin

SeLion's Code Generator Maven Plugin will auto-generate Java PageObject class files from PageYAML during the Maven generate-sources or generate-test-sourcesphase.

Benefits of Using the SeLion Code Generator Maven Plugin

If you have web pages that change frequently, this is your secret sauce to scaling your test efforts!

A PageYAML file contains all the information to interact with a page (or mobile screen). A change in your web page only requires a small change to YAML text. In addition, the PageYAML file allows you to define different identifiers based on locale.

Automating in different languages just got much easier!

If you're working across multiple teams, you can share the generated page classes as a packaged artifact. This allows other teams stay updated and does not impact test cases for every minor change.

Generated page classes can even extend other page classes for maximum reuse.

Configuration Parameters for Customizing the Code Generator Maven Plugin

  

  
    
    
  • <basePackage /> - This configuration parameter represents the base package name for the
      project.
      It defaults to com.paypal.selion.testcomponents. For example, if your PageYAML
      files are placed in src/main/resources/GUIData/mycompany then the corresponding generated
      java files will be placed in the package com.paypal.selion.testcomponents.mycompany. You can modify
      the <basePackage /> by adding a <configuration />
      section to the SeLion Code Generator plugin definition of your project pom.xml.
      
    
    
    • 
    
  • <excludeDomains /> - This configuration parameter represents the list of domains to exclude.
      Each domain is represented by a directory under
       src/main/resources/GUIData or src/test/resources/GUIData.
      
    
    
    • 
    
  • <htmlCustomElements /> - This configuration parameter represents the list of custom HTML elements
      that will be registered to the code generator. Each custom element should extend the class AbstractElement --
      make sure to implement all constructors.
     
    • 
     
  • <iosCustomElements /> - This configuration parameter represents the list of custom IOS elements
      that will be registered to the code generator. Each custom element should extend the class IOSPage --
      make sure to implement all constructors.
     
    • 
     
  • <androidCustomElements /> - This configuration parameter represents the list of custom Android elements
      that will be registered to the code generator. Each custom element should extend the class AndroidPage --
      make sure to implement all constructors.
     
    • 
     
  • <mobileCustomElements /> - This configuration parameter represents the list of custom Mobile elements
      that will be registered to the code generator. Each custom element should extend the class MobilePage --
      make sure to implement all constructors.

Put the SeLion Code Generator in Action

  
    
    
  1. First, create a PageYAML file that represents the actual page.
    2. 
    
  2. Once created, move the file into a directory under either
      src/main/resources/GUIData or src/test/resources/GUIData 
    3. 
    
  3. If you prefer to use the Maven command prompt, simply run the
      command mvn clean generate-sources or mvn clean generate-test-sources (if using
      test resoures for PageYaml) from your project directory.
    4. 
    
  4. If you would like to do this from Eclipse, right click your project
      and choose Maven > Update Project configuration.
    5. 
  


  
That's it! You now have the code generated, available to you as part of your build path, and
    you can start using the page object Java class.
