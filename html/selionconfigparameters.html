Configuration Parameters

SeLion provides rich set of configuration parameters for configuring its various features. Furthermore, configuration can be done via a TestNG formatted testng-suite.xml file, Java system properties, environment variables, or via code.

Configuration Parameter Scope

Configuring these parameters will override any default values. So, you should only specify the parameters you want to modify. Some configuration properties are global-only in scope which means any value set will apply to all tests executed in the test suite. Furthermore, you should not (and unless you insist, you can not) override any global-only value at the <test> level.

Parameter scope can apply to all test cases running from a TestNG <suite> or can apply to a particular <test> in the <suite>.

  • Parameters defined at the <suite> level are called global or per <suite> parameters.
  • Parameters defined at the <test> level are called local or per <test> parameters.
  • Any configuration parameters specified for a <test> apply to all test methods of that <test> and will override any global <suite> values defined unless the configuration property is global-only

Below are the configuration parameters for SeLion. Parameters that are global-only are noted as GLOBAL in the scope column. Parameters which can be configured to both global and local are defined as GLOBAL, LOCAL.


TestNG Suite file config value JVM argument config value Default Value Scope Description
autoScreenShot SELION_AUTO_SCREEN_SHOT true GLOBAL Automatically take screen shots.
seleniumhost SELION_SELENIUM_HOST IP Address of the machine wherein a Grid is running. GLOBAL

Selenium host might be localhost or other where server is running. Used when Config Parameter SELENIUM_RUN_LOCALLY is false.
seleniumport SELION_SELENIUM_PORT 4444 GLOBAL

Selenium port, any port where Selenium is running. Default is set to 4444. Used when Config Parameter SELENIUM_RUN_LOCALLY is false.
firefoxProfile SELION_SELENIUM_FIREFOX_PROFILE   GLOBAL, LOCAL

The firefox profile that is to be used for local/remote runs. Use this when you would like to have firefox work with your custom firefox profile. You can either specify the name of the profile or you can specify the directory location of your firefox
profile. The framework internally first assumes it to be a profile name. If that assumption fails we fail-over to assuming it to be the firefox profile directory. If that also fails we will work with an anonymous firefox profile. Please be aware that if your firefox profile directory is very huge (above ~10 MB) then there are chances that it can crash your local JVM.
saucelabGridConfigFile SELION_SELENIUM_SAUCELAB_GRID_CONFIG_FILE   GLOBAL

The name of the JSON file that contains the configuration customization info needed for saucelabs. By default, the configurations specified at Sauce labs Configurations are applicable.

chromeDriverPath SELION_SELENIUM_CHROMEDRIVER_PATH   GLOBAL

The path to the chromedriver executable on the local machine. This parameter is taken into consideration for local runs involving googlechrome browser alone.
phantomjsPath SELION_SELENIUM_PHANTOMJS_PATH   GLOBAL

The path to the PhantomJS executable on the local machine. This parameter is taken into consideration for local runs involving PhantomJS browser alone. The binary can be downloaded from http://phantomjs.org/download.html.
microsoftWebDriverPath SELION_SELENIUM_EDGEDRIVER_PATH   GLOBAL

The path to the MicrosoftWebDriver executable on the local machine. This parameter is taken into consideration for local runs involving Edge browser alone. The MicrosoftWebDriver.msi file can be downloaded from https://www.microsoft.com/en-us/download/confirmation.aspx?id=48740.
userAgent SELION_SELENIUM_USERAGENT   GLOBAL, LOCAL

Use this parameter to set the user agent for firefox when working with Mobile version. This parameter should be set in conjunction with the parameter BROWSER.
proxyServerHost SELION_SELENIUM_PROXY_HOST   GLOBAL Use this parameter to set the Proxy server name to be used.
proxyServerPort SELION_SELENIUM_PROXY_PORT   GLOBAL Use this parameter to set the Proxy server port to be used.
useSauceLabGrid SELION_SELENIUM_USE_SAUCELAB_GRID false GLOBAL, LOCAL

Use this parameter to indicate if your remote runs are to be run against the sauce lab grid or against the QI owned grid/your own grid. This flag is required because when running against Sauce lab furnished grid, we are to ensure that fetching of WebDriver node IP and Port is to be disabled.
customCapabilities SELION_SELENIUM_CUSTOM_CAPABILITIES_PROVIDER   GLOBAL, LOCAL

Use this parameter to provide SeLion with a custom capabilities provider. The value for this parameter would be the fully qualified class name which is a sub-class of DefaultCapabilitiesBuilder. If more than one custom capabilities providers are required, please separate the fully qualified class names with commas.
webDriverEventListener SELION_SELENIUM_WEBDRIVER_EVENT_LISTENER   GLOBAL

Use this parameter to provide SeLion with a custom listener which can be plugged into EventFiringWebDriver. If the fully qualified class implements WebDriverEventListener then SeLion will invoke the custom implementation provided by you via your implementation as and when the relevant events happen. If more than one custom listeners are required, please separate the fully qualified class names with commas.
runLocally SELION_SELENIUM_RUN_LOCALLY false GLOBAL

Flip this parameter to true if you would like a browser to be spawned locally on your machine and run automation tests there. Default is set to false which means your tests are always going to be run against a Remote Grid environment as pointed to by Config Parameter SELENIUM_HOST.
mobileAppFolder SELION_MOBILE_APP_FOLDER "Applications" folder in working directory GLOBAL

This parameter represents the folder which would contain the IOS app. This parameter is currently only useful for local runs. This is the folder from which applications would be searched for by SeLion when it comes to iOS automation in local runs. So please ensure that all the built applications reside in this folder. This is a mandatory value without which iOS native automation would not work. If no value is provided, SeLion assumes that the application is available under a directory named "Applications" in the current working directory.
mobileAppName SELION_MOBILE_APP_NAME   GLOBAL, LOCAL

This parameter represents the name of the app that is to be spawned. Merely specifying the name of the native app should suffice.
mobileAppLanguage SELION_MOBILE_APP_LANGUAGE en GLOBAL, LOCAL

This parameter represents the language to be used. By default it is always English represented as en
mobileAppLocale SELION_MOBILE_APP_LOCALE en_US GLOBAL, LOCAL

This parameter represents the locale to be used. By default it is always US English represented as en_US.
appPath SELION_MOBILE_APP_PATH   GLOBAL, LOCAL

This parameter represents the fully qualified path of the app that is to be spawned. Must be one of an absolute path on the local filesystem, or a http:// URL, a file on "sauce storage" (sauce-storage:testApp.apk), or a file on "SeLion hub storage" (selion-hub-storage:uuid/testApp.apk).
mobileNodeType SELION_MOBILE_NODE_TYPE   GLOBAL, LOCAL

This parameter represents the mobile driver which is used to execute mobile test cases. The valid values are appium, ios-driver, selendroid
sauceUserName SELION_SAUCELAB_USER_NAME   GLOBAL This parameter is used to pass SauceLabs user name
sauceApiKey SELION_SAUCELAB_API_KEY   GLOBAL This parameter is used to pass SauceLabs api key
GUIDataDir SELION_GUI_DATA_DIR GUIData GLOBAL

Represents the directory from wherein resource files which houses locators for generating SeLion PageObjects.
siteLocale SELION_SITE_LOCALE US GLOBAL, LOCAL

Site will show country used for tests.
browser SELION_BROWSER *firefox GLOBAL, LOCAL Browser specified by user.
version SELION_BROWSER_CAPABILITY_VERSION   GLOBAL, LOCAL version specified by user when working with custom browser needs.
enableGUILogging SELION_ENABLE_GUI_LOGGING false GLOBAL

Turn this flag ON to see GUI actions such as loading a URL, click/setting text etc., being logged into the test reports that get generated by SeLion.
baseDir SELION_BASE_DIR selionFiles GLOBAL The base directory for SeLion files.
workDir SELION_WORK_DIR BASE_DIR.defaultValue + "/selionWorkDir" GLOBAL The work directory of SeLion
platform SELION_BROWSER_CAPABILITY_PLATFORM ANY GLOBAL, LOCAL platform is specified by user. Supporting values are: ANDROID, ANY, LINUX, MAC, UNIX, VISTA, WINDOWS, XP.
enableJavaScript SELION_BROWSER_CAPABILITY_SUPPORT_JAVASCRIPT true GLOBAL, LOCAL

Should javascript capability be enabled on the browser for the AUT. By default javascript will be enabled on the client browser, but this flag can be used to toggle this setting.
browserHeight SELION_BROWSER_HEIGHT   GLOBAL, LOCAL

Set the Height of the Browser Window Should be a whole number.
browserWidth SELION_BROWSER_WIDTH   GLOBAL, LOCAL

Set the Width of the Browser Window Should be a whole number.
executionTimeout SELION_EXECUTION_TIMEOUT 120000 milli seconds GLOBAL, LOCAL

Timeout for an execution command, in milliseconds. Used in SeLion to configure Selenium timeouts
logPages SELION_LOG_PAGES true GLOBAL

Automatically log pages source code. Used in conjunction with ConfigProperty autoScreenShot
printDebugInfo SELION_PRINT_DEBUG_INFO true GLOBAL

To print debug info (about Browser/Selenium Version/Client OS) for user.
mobileDevice MOBILE_DEVICE "" GLOBAL

mobile device to be used for running tests. The structure is device:platform version. ex: iPhone 6:9.3, nexus5:6.0. device parameter will be override if mobileDeviceType is defined
mobileDeviceType MOBILE_DEVICE_TYPE "" GLOBAL

mobile device type to be used for running tests.

Configuring Parameters as JVM Command Line Arguments

  
Configuration parameters can be specified as Java system properties on the command line.
    For example:
  
mvn test -DSELION_SELENIUM_RUN_LOCALLY=true -DSELION_BROWSER="*firefox"


  
Configuration parameters specified as Java system properties or environment variables will override any values
    specified for the same configuration setting in the TestNG suite.
